git‑ents

An editorial Git forge for people who love to read their code.

build passing v0.7.0 MIT rust 1.78+

git-ents renders a repository as quiet, server-side HTML — a README that reads like a page, a file tree you can wander, releases as a changelog, and CI defined entirely with the Git hooks you already know. The whole UI is paper-grain and works without a single byte of client JavaScript.

Why git‑ents

  • Readable by default. Lora display type, generous measure, warm paper palette — light and dark are equals.
  • Hooks, not pipelines. CI is just pre-receive and post-receive scripts, with run history and logs.
  • No JavaScript. Every page is fully server-rendered; the browser just paints.
  • One binary. Point it at a directory of bare repos and serve.

Install

$ cargo install git-ents
$ git-ents serve ./repos # http://localhost:8420

A taste of the renderer

// render any blob as warm, highlighted HTML
pub fn render_blob(blob: &Blob) -> Markup {
    let lang = Language::guess(&blob.path);
    html! {
        pre.paper { (highlight(&blob.text, lang)) }
    }
}
